blend_overlay(a, b)

source code  
Returns either multiply or screen for darker or lighter values of A
(the inverse of hard light)
C =
   A < 0.5 : 2 * A * B
   A >=0.5 : 1 - (2 * (255-A) * (255-B))

blend_hard_light(a, b)

source code  
Returns either multiply or screen for darker or lighter values of B
(the inverse of overlay)
C =
  B < 0.5 : 2 * A * B
  B >=0.5 : 1 - (2 * (255-A) * (255-B))

blend_dodge(a, b)

source code 

Returns the first (underlay) color divided by the inverse of the second (overlay) color. C = A / (255-B)

blend_burn(a, b)

source code 

Returns the inverse of the inverse of the first (underlay) color divided by the second (overlay) color. C = 255 - (255-A) / B

blendColor(c1, c2, MODE)

source code 

Implements the blending of two colors. MODE is one of the blend mode constants defined in pyprocessing (an integer between 0 and 13). This expects colors expressed as integers.
